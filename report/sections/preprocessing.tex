\section{Препроцессинг данных}

Проводится два эксперимента с разными условиями подготовки данных.

\subsection{Эксперимент 1 --- базовый препроцессинг}

\textbf{Токенизация.} Каждое предложение токенизируется независимо с помощью HuggingFace AutoTokenizer (\texttt{is\_split\_into\_words=True}). Поскольку WordPiece и BPE разбивают слова на субтокены, метки выравниваются по правилу: первый субтокен слова получает метку слова, последующие субтокены --- метку \texttt{I-X} (если слово начинается с \texttt{B-X}) или ту же метку \texttt{I-X}. Специальные токены \texttt{[CLS]}, \texttt{[SEP]} получают маску \texttt{-100} и не участвуют в вычислении функции потерь.

\textbf{Динамический паддинг.} Паддинг выполняется не при токенизации, а при сборке батча: последовательности дополняются до длины самой длинной в батче. Это уменьшает вычислительную нагрузку по сравнению с паддингом до фиксированной длины 512.

Аугментация и межпредложенческий контекст не применяются.

\subsection{Эксперимент 2 --- улучшенный препроцессинг}

Все изменения касаются только обучающей выборки; валидационная и тестовая обрабатываются так же, как в эксперименте 1 (кроме контекстного окна).

\subsubsection{Аугментация заменой сущностей}

Строится словарь сущностей по типу: для каждого типа собираются все span'ы из обучающей выборки. Для каждого предложения каждый span заменяется с вероятностью $p = 0.5$ случайным span'ом того же типа из словаря. Предложения, в которых произошла хотя бы одна замена, добавляются в обучающую выборку как новые примеры (оригиналы сохраняются). Метод позволяет модели видеть больше контекстов для каждого типа сущности и снижает переобучение на конкретные упоминания.

\subsubsection{Межпредложенческий контекст}

Для каждого предложения к нему приписываются соседние предложения из того же документа (окно $w = 1$): одно предшествующее и одно следующее. Токены контекстных предложений получают метку-заглушку, которая отображается в маску \texttt{-100}, поэтому функция потерь вычисляется только по центральному предложению. BERT при этом кодирует токены с учётом соседнего контекста, что даёт модели более богатые представления на границах предложений.

Схема для предложения $i$ в документе $[s_1, \ldots, s_n]$:
\[
    \underbrace{s_{i-1}}_{\text{контекст}} \; \underbrace{s_i}_{\text{цель}} \; \underbrace{s_{i+1}}_{\text{контекст}}
\]

\subsection{Метрика оценки}

Качество оценивается по entity-level macro F1 (библиотека \texttt{seqeval}). Сущность считается правильно распознанной только при точном совпадении границ span'а \textit{и} типа. Macro F1 усредняет F1 по всем 6 типам с равными весами, что делает метрику чувствительной к редким классам.
